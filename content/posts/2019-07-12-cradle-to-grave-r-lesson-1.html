Open RStudio

Once RStudio is open your screen will look like this: The left-hand-side of the screen will be the Console Area, and inside that, you‚Äôll see a greater-than sign > followed by a blinking cursor. This is where you‚Äôll input code. On the top-right hand side, you‚Äôll find the Environment. At the start, this will be empty because nothing has been stored there, but later on, this is where your variables will be stored.

In the bottom-right of the screen, you‚Äôll find several other tabs, and usually the Packages or Files tab is selected by default.

Getting Started

You can go right ahead and type simple calculations into the console. You can try typing 2 + 4 then hitting ENTER and you‚Äôll get the result [1] 6. What this means is that the first line of your output [1] has the result of 6. Carry on and follow the other examples in the video including multiplications, exponents, trigonometric functions, and more.

This is a great start. However, we don‚Äôt really want to work in the console, as this is a bit like working directly on a stove. We don‚Äôt want it all to cook immediately. Instead, we need to create a recipe first and we‚Äôre going to call this recipe an ‚ÄòR Script‚Äò. We can store these recipes just like you would a cooking book, and not actually cook anything until we‚Äôre ready to put it all on the stove.

## Create Your First R Script

To create your first R Script go to the File menu, then New File, then R Script. A new window will appear above the console, with the tab Untitled 1. This will be your script. It‚Äôs a great habit to save at this point and regularly as you go.

Scripts offer the simplest form of repeatable analysis in R. They‚Äôre text files that contain codes and comments, and their extension is .R (dot with a capital R).

A great practice with scripts is to leave comments. You can do this by starting any line with the # symbol. Anything to the right of that hashtag or pound symbol will not be executed when you run the script.

Down the left-hand side of the script, you‚Äôll see the line numbers 1, 2, 3 etc.

At this point, if you want to change the look and feel of RStudio, head to the Tools menu, followed by Global Options, and then go to the Appearance tab. From there you can change font, text size, zoom and you can choose an Editor Theme. The themes change the general look of a few different parts of the program all at once, so click through a few and choose the one you like the look of.

Once you‚Äôre happy with the look of your RStudio, go ahead and try entering a basic math command in the script window such as 2 + 2 as in the video. You‚Äôll notice that when you hit ENTER, nothing happens! This is different from the console window, where the formula would be calculated immediately. If you look lower on the screen, you‚Äôll see you still have the console window available, it‚Äôs just moved to the bottom.

To get the formula to calculate in the console window, you‚Äôll need to hit Cmd-Enter on the Mac, or Ctrl+Enter on the PC when in the script window. Then if you look below, you‚Äôll see the results posted in the console window.

Assign a Value to a Variable

To assign a value to a variable in RStudio, go to the script window and change your 2 + 2 line to read x <- 2 + 2
When you hit Cmd/Ctrl+Enter you‚Äôll then notice in the global environment window (that‚Äôs on the top-right of your screen) shows X being equal to 4. From now on if I want to use the value x in the console window, the program will automatically know that it‚Äôs a 4. Try typing x then hitting enter in the console window and you‚Äôll see the result is 4. Try the same thing in the script window, but remember to use cmd/ctrl+enter.

Remember to save your work regularly. At this point let‚Äôs save what you‚Äôve created. Go to the File menu > Save As‚Ä¶ then name and save your progress. Once you‚Äôve done this, the name of the tab above the script window will change from ‚ÄòUntitled‚Äô to reflect your new file name, finished by .R at the end.

Run Your Script

Now it‚Äôs time to run your script! You‚Äôll find a Run button along the top of the script window. It performs a similar function to pressing cmd/ctrl+enter. Meaning that if you hit the run button, it will only run the current line that your blinking cursor is on. If your cursor is on line 5, it will run only line 5, for example. If you want to run the entire script you‚Äôll first need to use your mouse to highlight everything you want to run, and then press the Run button.

Notice that when you highlight the lines starting with # they don‚Äôt affect the output. This is because they‚Äôre just comments and won‚Äôt be a functioning part of the code.

Next up you can make your script a little more complex, and create another variable using your first. Try changing line 6 to y <- x + 4 then running that line. Take a look in the Global Environment window and if you‚Äôve done everything right you‚Äôll see the new variable and it‚Äôll be 4 more than x. Note that in the video I changed the value for x a little earlier too, so if your numbers don‚Äôt look the same don‚Äôt worry too much, you can alter your code to match what I have in the script window. These variables now remain in the computer memory for us to use any time we need to, until we close RStudio.

Summary

Now you‚Äôre able to make, run and save an R Script, which is good for some simple analysis.

If you want to publish this with documentation, methods, conclusion etc then R Script isn‚Äôt the best method for that, so in the next lesson, we‚Äôre going to look at R Markdowns and R Notebooks to help with that.
