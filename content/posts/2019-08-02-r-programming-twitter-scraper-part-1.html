Register for a Twitter API

Before interacting with Twitter you need to register for a Twitter API. To do this go to apps.twitter.com – this is completely free, simple and only takes a couple of minutes to do.

If you’re not registered on the site you’ll need to go ahead and complete that part and sign in. Once that’s done, you should see a screen similar to the one above, except that you won’t have any apps like my “TrumpGrabber” there. Click on Create New App, and you’ll see this screen next:

For Name and Description you can make up your own, anything you like. In the Website section, the small print says that you don’t have to have a website, so just fill this in with something as a place holder for now, and we can come back and change it later. You don’t need to fill in the Callback URL.

Then hit Create your Twitter application down at the bottom of the page and this is going to give you access to an API Key, ****a Secret API Key and Token Keys. That part should be pretty self-explanatory.

Using Your API Keys in R Studio

Switch to R Studio and we’ll figure out how to use your API keys in your code.

Following the example in the screenshot above, paste your API codes in the relevant spots on lines 5-9 replacing the XXXXXs with the codes. Copy the API Key from Twitter to the API Key line in the code. Copy the API Secret from Twitter to the API Secret line in the code. The hardest part of this is getting the Access Token which you’ll need to grab by clicking generate token on the Twitter account.

Install Libraries

There are two libraries we need to install, twitteR and ROAuth, both shown in the screenshot above. You can include these in the code, but you might have to install them.

To do so, go to the Packages tab in the lower-left and then the Install tab. In the pop-up search the name of the first library by starting to type twitteR and as you type the list will shorten until you find the library you want. Capitals do matter here so make sure to get them right. Hit install when you’re done and repeat for ROAuth.

Grab Some Tweets Using the Code

Line 11 is where the magic starts to happen and you connect to the Twitter API. This line instructs the program to set up the Twitter OAuth, enter the API key and secret, access token and access token secret.

Lines 14-16 describe to the program what tweets to grab from Twitter.

Line 14 asks for the last 50 (n=50) tweets from the username realDonaldTrump and assigns these to the variable tweets_trump.

Line 15 asks for the last 200 tweets containing @realDonaldTrump and assigns them to tweets_trump2.

Line 16 looks for the last tweets containing morning+bacon and assigns them to tweets_trump3.

It’s important to know that there’s a limit to how far back you can go with the Twitter API. At some point in the last week or ten days or so, you won’t be able to grab any more data.

Place the Tweets into Data Frames

Go ahead and run lines 19-21 and this will assign the data in our variables into data frames shown in the Global Environment. Click on tweets.df to bring up the list of data contained inside.

On the left-hand side, you’ll see the various tweets sorted into rows and columns including the text of the tweet, how many times it’s been favorited when it was created, and so on. The reason we use data frames is that the data is easier to manipulate in this form compared to just using the variables such as tweets_trump where all the data is more compacted and not so useful.

Write Your Data Frames to CSV

Lastly, you can write the data to comma separated value files (CSV) by using the code write.csv and then specifying the variables as per lines 23-25. This way you’ll be able to bring up the data in Excel or Google Sheets too.
